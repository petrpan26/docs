---

All errors return JSON with a structured `detail` field containing an error code:

```json
{
  "detail": {
    "error_code": "CS-ORIV-001",
    "message": "Reference image is required",
    "action": "Upload a ref_image (PNG/JPG)"
  }
}
```

Transient errors include a `retryable: true` field — your client should automatically retry these:

```json
{
  "detail": {
    "error_code": "CS-ORGS-001",
    "message": "Failed to upload to storage",
    "action": "Retry the job",
    "retryable": true
  }
}
```

When contacting support, always include the `error_code` from the response. Email **support@viggle.ai** with the error code and your job ID for fastest resolution.

## Error response fields

| Field | Description |
|-------|-------------|
| `error_code` | Unique identifier (e.g. `CS-ORIV-001`). Always include this when contacting support. |
| `message` | Human-readable description of what went wrong |
| `action` | What you can do to fix it |
| `retryable` | If `true`, the error is transient and the request can be retried |
| `detail` | Additional context (not always present) |

## Error codes by category

### Authentication (401)

| Code | Message | Action |
|------|---------|--------|
| `CS-ORAU-001` | API key is required | Include `Authorization: Bearer sk-...` header |
| `CS-ORAU-002` | API key is invalid or not found | Check your API key is correct |
| `CS-ORAU-003` | API key has expired | Generate a new API key in the [dashboard](https://portal.viggle.ai/dashboard) |

### Credits (402)

| Code | Message | Action |
|------|---------|--------|
| `CS-ORCR-001` | Not enough credits for this job | Top up your credits in the [dashboard](https://portal.viggle.ai/dashboard) |

### Input validation (400)

| Code | Message | Action |
|------|---------|--------|
| `CS-ORIV-001` | Reference image is required | Upload a `ref_image` (PNG/JPG) |
| `CS-ORIV-002` | Driving video is required | Upload a `driving_video` (MP4) |
| `CS-ORIV-003` | Video format not supported | Upload MP4, MOV, or AVI format |
| `CS-ORIV-004` | Video exceeds maximum duration | Keep video under 5 minutes |
| `CS-ORIV-005` | Video file too large | Keep file under 500MB |
| `CS-ORIV-006` | Video resolution too high | Keep resolution under 4096x4096 |
| `CS-ORIV-007` | Could not read video file | Ensure the file is a valid video |
| `CS-ORIV-008` | Character not found | Check the `character_id` exists |
| `CS-ORIV-009` | Scene not found | Check the `scene_id` exists |
| `CS-ORIV-010` | Scene preprocessing not complete | Wait for scene status to be `ready` before rendering |
| `CS-ORIV-011` | Invalid background mode | Use `original`, `solid`, or `transparent` |
| `CS-ORIV-012` | Invalid character_mapping JSON | Provide valid JSON for multi-character mapping |
| `CS-ORVV-002` | Video has no frames | Upload a valid video with at least 1 frame |
| `CS-ORVV-003` | Video FPS unreadable | Re-encode the video with a standard frame rate |

### Job status (404 / 410)

| Code | Message | Retry | Action |
|------|---------|:-----:|--------|
| `CS-ORJC-001` | Job not found | | Check the job ID is correct |
| `CS-ORJC-003` | Result has expired | | Submit a new job — results expire after 24 hours |
| `CS-ORJC-005` | Job failed during processing | Yes | Retry the job. If persistent, contact support. |

### Processing errors (500)

| Code | Message | Retry | Action |
|------|---------|:-----:|--------|
| `CS-ENEC-001` | Encoder could not access the video | Yes | Retry the job |
| `CS-ENEC-004` | No humans detected in some frames | | Ensure the person is visible throughout the video |
| `CS-ENEC-006` | GPU out of memory during encoding | Yes | Try a lower resolution or shorter video |
| `CS-RNRD-003` | GPU out of memory during rendering | Yes | Try a lower resolution video |
| `CS-MGMR-001` | Some video chunks are missing | Yes | Retry the job |

### Capacity (503)

| Code | Message | Retry | Action |
|------|---------|:-----:|--------|
| `CS-ORRS-001` | All workers are busy | Yes | Retry in a few seconds or use normal mode instead of fast mode |

</Warning>

## HTTP status codes

| Status | Meaning | Error codes |
|--------|---------|-------------|
| 400 | Bad Request — fix your request | `CS-ORIV-*`, `CS-ORVV-*` |
| 401 | Unauthorized — check API key | `CS-ORAU-*` |
| 402 | Payment Required — top up credits | `CS-ORCR-001` |
| 404 | Not Found | `CS-ORJC-001` |
| 410 | Gone — result expired | `CS-ORJC-003` |
| 500 | Server Error — retry, forward to support if persistent | `CS-ENEC-*`, `CS-INIP-*`, `CS-RNRD-*`, `CS-MGMR-*`, `CS-ORPH-*`, `CS-ORSH-*`, `CS-ORTO-*`, `CS-ORGS-*`, `CS-ORRD-*` |
| 503 | Unavailable — retry later | `CS-ORRS-001` |

## Error handling

<AccordionGroup>
  <Accordion title="Implement retry logic">
    Check the `retryable` field in error responses to decide whether to retry:

    ```python
    import time, requests

    def api_request_with_retry(url, headers, max_retries=3):
        for attempt in range(max_retries):
            response = requests.get(url, headers=headers)
            if response.status_code >= 400:
                data = response.json()
                detail = data.get("detail", {})
                if isinstance(detail, dict) and detail.get("retryable"):
                    time.sleep(2 ** attempt)  # exponential backoff
                    continue
                return response  # not retryable, return immediately
            return response
        raise Exception("Max retries exceeded")
    ```
  </Accordion>
  <Accordion title="Handle error codes">
    Check for error codes in the response:

    ```python
    response = requests.post(url, headers=headers, files=files)
    if response.status_code != 200:
        error = response.json().get("detail", {})
        code = error.get("error_code", "unknown")
        message = error.get("message", response.text)
        action = error.get("action", "")
        print(f"Error {code}: {message}")
        print(f"Fix: {action}")
        # Forward to support: support@viggle.ai with the error_code
    ```
  </Accordion>
  <Accordion title="Handle processing failures">
    Always check for failed status and error codes when polling. Internal errors (`CS-ORPH-*`, `CS-ORSH-*`, `CS-ORTO-*`) should be retried automatically and escalated to support if they persist:

    ```python
    INTERNAL_ERRORS = {"CS-ORPH-001", "CS-ORPH-002", "CS-ORSH-001", "CS-ORSH-999", "CS-ORTO-001"}

    while True:
        status = get_status(job_id)
        if status["status"] == "complete":
            break
        elif status["status"] == "failed":
            code = status.get("error_code", "unknown")
            error = status.get("error", "")
            print(f"Job failed: {code} - {error}")

            if code in INTERNAL_ERRORS:
                # Internal error — safe to retry
                print("Internal error. Retrying...")
                job_id = submit_new_job()  # retry
            else:
                # User error — fix the request
                print(f"Fix: check the error code at https://docs.viggle.ai/error-codes")
            break
        time.sleep(5)
    ```
  </Accordion>
  <Accordion title="Validate before rendering">
    Check that assets are ready before creating render jobs:

    ```python
    character = get_character(character_id)
    scene = get_scene(scene_id)
    if character["status"] != "ready":
        raise Exception("Character not ready — wait for status 'ready'")
    if scene["status"] != "ready":
        raise Exception("Scene not ready — wait for status 'ready'")
    create_render_job(character_id, scene_id)
    ```
  </Accordion>
</AccordionGroup>

## Need help?

If you encounter an error that persists after retrying, email **support@viggle.ai** with:
1. The `error_code` from the response
2. Your job ID (`job_id`)
3. The timestamp of the error

<Tip>
**Internal errors** (codes starting with `CS-ORPH`, `CS-ORSH`, or `CS-ORTO`) are infrastructure issues on our side. You don't need to change your request — just retry. If the same internal error occurs 3+ times, please forward the error code and job IDs to **support@viggle.ai** so our engineering team can investigate the root cause.
</Tip>
