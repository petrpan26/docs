---
title: 'Render Options'
description: 'Configure video generation settings'
---

Configure video generation by passing form fields when creating a render job via `POST /api/render` (multipart/form-data). All options are optional with sensible defaults.

## Available Options

<ParamField body="model" type="string" default="V4">
  The rendering model. Currently only `V4` is available.
</ParamField>

<ParamField body="background_mode" type="string" default="inpaint">
  Background handling mode.

  | Value | Description |
  |-------|-------------|
  | `inpaint` | AI fills the background intelligently (default) |
  | `original` | Keep the original video background |
  | `solid` | Replace with solid color (use with `bg_color`) |
  | `transparent` | Transparent background with separate mask video for compositing |
</ParamField>

<ParamField body="bg_color" type="string" default="0,255,0">
  RGB string for solid background color. Only used when `background_mode` is `solid`. Format: `"R,G,B"`.

  ```
  bg_color=0,255,0       # Green screen
  bg_color=0,0,0         # Black
  bg_color=255,255,255   # White
  ```
</ParamField>

<ParamField body="resolution" type="string" default="original">
  Output resolution. Use `original` to match the input, or specify a target like `720p` or `1080p`.

  | Value | Description |
  |-------|-------------|
  | `original` | Match input video resolution (default) |
  | `720p` | 1280x720 |
  | `1080p` | 1920x1080 |
</ParamField>

<ParamField body="fast" type="boolean" default="false">
  Enable fast mode for faster rendering.

  <Warning>
    Fast mode requires available capacity. If capacity is unavailable, you'll receive a `503` error. Either retry with `fast=false` or wait for capacity.
  </Warning>
</ParamField>

## Example Usage

### Default Options

```bash
curl -X POST "https://api.viggle.ai/api/render" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d "character_id=char_xxx" \
  -d "scene_id=scene_xxx"
```

Uses default inpaint background.

### Green Screen

```bash
curl -X POST "https://api.viggle.ai/api/render" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d "character_id=char_xxx" \
  -d "scene_id=scene_xxx" \
  -d "background_mode=solid" \
  -d "bg_color=0,255,0"
```

### Fast Processing

```bash
curl -X POST "https://api.viggle.ai/api/render" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d "character_id=char_xxx" \
  -d "scene_id=scene_xxx" \
  -d "fast=true"
```

### Transparent for Compositing

```bash
curl -X POST "https://api.viggle.ai/api/render" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d "character_id=char_xxx" \
  -d "scene_id=scene_xxx" \
  -d "background_mode=transparent"
```

### Custom Resolution

```bash
curl -X POST "https://api.viggle.ai/api/render" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d "character_id=char_xxx" \
  -d "scene_id=scene_xxx" \
  -d "resolution=1080p"
```

## Background Mode Comparison

| Mode | Use Case | Output |
|------|----------|--------|
| `inpaint` | General use, social media | AI-generated background that matches the scene |
| `original` | Keep scene context | Original video background preserved |
| `solid` | Green screen, compositing | Solid color background for easy keying |
| `transparent` | Professional compositing | Black background video + separate mask video for layering in video editors |

## Best Practices

<AccordionGroup>
  <Accordion title="Background selection">
    - Use **inpaint** when you want a complete, ready-to-share video
    - Use **original** when the scene context is important
    - Use **solid** or **transparent** when compositing in video editors
  </Accordion>

  <Accordion title="Transparent mode and mask">
    When using `background_mode=transparent`, the API generates two outputs:
    - **Main video**: Character rendered on a black background
    - **Mask video**: Grayscale video (white = character, black = background)

    Use both files in your video editor to composite the character onto any background. The mask download URL is returned in the job status response as `mask_cdn_url`.
  </Accordion>

  <Accordion title="Performance optimization">
    - Enable **fast** mode for time-sensitive renders when capacity is available
    - Use **720p** resolution for faster renders when full resolution isn't needed
  </Accordion>
</AccordionGroup>
