title: 'Render Options'

Pass optional form fields when creating a render job via `POST /api/render`. All options have sensible defaults.

## Available options

<ParamField body="model" type="string" default="V4_Preview">
  The rendering model. Currently only `V4_Preview` is available. New models will be added in future releases.
</ParamField>

<ParamField body="background_mode" type="string" default="original">
  Background handling mode.

  | Value | Description |
  |-------|-------------|
  | `original` | Keep the original video background (default) |
  | `solid` | Replace with solid color (use with `bg_color`) |
  | `transparent` | Character on black background + separate grayscale mask video for compositing |
</ParamField>

<ParamField body="bg_color" type="string" default="0,255,0">
  RGB string for solid background. Only used when `background_mode` is `solid`. Format: `"R,G,B"` (0-255, not hex).
</ParamField>

<ParamField body="resolution" type="string" default="SD">
  Output resolution. `SD` for standard definition, `HD` for high definition.
</ParamField>

<ParamField body="fast" type="boolean" default="false">
  Enable fast mode for faster rendering.

  <Warning>Fast mode requires available capacity. Returns `503` if unavailable — retry with `fast=false` or wait.</Warning>
</ParamField>

    ```bash
    curl -X POST "https://apis.viggle.ai/api/render" \
      -H "Authorization: Bearer YOUR_API_KEY" \
      -F "character_id=char_xxx" -F "scene_id=scene_xxx" \
      -F "background_mode=solid" -F "bg_color=0,255,0"
    ```
  </Tab>
  <Tab title="Transparent" icon="scissors">
    Character on black background + mask video for compositing.

    ```bash
    # 1. Submit render
    curl -X POST "https://apis.viggle.ai/api/render" \
      -H "Authorization: Bearer YOUR_API_KEY" \
      -F "character_id=char_xxx" -F "scene_id=scene_xxx" \
      -F "background_mode=transparent"

    # 2. Poll — response includes both cdn_url and mask_cdn_url
    curl "https://apis.viggle.ai/api/render/JOB_ID/poll" \
      -H "Authorization: Bearer YOUR_API_KEY"
    # { "status": "complete", "cdn_url": "...", "mask_cdn_url": "..." }

    # 3. Download both
    curl -L -o character.mp4 "CDN_URL"
    curl -L -o mask.mp4 "MASK_CDN_URL"
    ```

    See [Compositing Guide](/guides/compositing) for full workflow.
  </Tab>
  <Tab title="Fast" icon="zap">
    Priority rendering when capacity is available.

    ```bash
    curl -X POST "https://apis.viggle.ai/api/render" \
      -H "Authorization: Bearer YOUR_API_KEY" \
      -F "character_id=char_xxx" -F "scene_id=scene_xxx" \
      -F "fast=true"
    ```
  </Tab>
  <Tab title="On-Demand" icon="upload">

      -H "Authorization: Bearer YOUR_API_KEY" \

## Background mode comparison

| Mode | Output | `mask_cdn_url` | Use Case |
|------|--------|:---------:|----------|
| `original` | Character composited on original video background | No | General use |
| `solid` | Character on solid color (e.g. green screen) | No | Chroma keying |
| `transparent` | Character on black + grayscale mask video | **Yes** | Professional compositing (AE, Resolve, FFmpeg) |

## Transparent mode details

When `background_mode=transparent`:

1. **Main video** (`cdn_url`): Character rendered on a black background
2. **Mask video** (`mask_cdn_url`): Grayscale video where white = character, black = background

Both URLs are included in the poll response when the job completes. Use them together in your video editor as a luma matte.

```json
{
  "status": "complete",
  "cdn_url": "https://assets.viggle.ai/.../output.mp4",
  "mask_cdn_url": "https://assets.viggle.ai/.../output_mask.mp4",
  "completed_chunks": 17,
  "total_chunks": 17
}
```

<Info>
  For step-by-step compositing instructions (After Effects, DaVinci Resolve, FFmpeg), see the [Compositing Guide](/guides/compositing).
</Info>

## Best practices

<AccordionGroup>
  <Accordion title="Background selection">
    - **`original`** — keep scene context, easiest to use
    - **`solid`** — traditional green screen, works with any chroma key tool
    - **`transparent`** — most flexible, gives you the mask for precise compositing
  </Accordion>
  <Accordion title="On-demand vs preprocessed">
    - **On-demand** — upload `ref_image` + `driving_video` directly. Simplest but slower (encoding + rendering in one step).
    - **Preprocessed** — preprocess `character_id` and `scene_id` separately, then combine. Faster rendering since encoding is done ahead of time.
  </Accordion>
  <Accordion title="Performance">
    Enable **fast** mode for time-sensitive renders when capacity is available. Videos are automatically resampled to 25fps for consistent output.
  </Accordion>
</AccordionGroup>
