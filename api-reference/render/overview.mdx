The Render API combines a character and scene to generate an animated video. Use the **Poll endpoint** with long-polling to receive video chunks progressively as they render, or check the **Status endpoint** and download the complete video when ready.

## Job Status Flow

Jobs progress through the following statuses:

```mermaid
graph LR
    A[queued] --> B[processing]
    B --> C[rendering]
    C --> D[merging]
    D --> E[complete]
    C --> F[failed]
    B --> F
```

| Status | Description |
|--------|-------------|
| `queued` | Job is waiting in the queue |
| `processing` | Job is being prepared |
| `rendering` | Video frames are being generated |
| `merging` | All chunks rendered, final video is being assembled |
| `complete` | Video is ready for download |
| `failed` | Job failed (check `error` field) |

## Live Render Example

```python
# Long-poll for chunks as they render
last_chunk = -1
while True:
    resp = requests.get(
        f"{BASE}/api/render/{job_id}/poll",
        headers=headers,
        params={"last_chunk": last_chunk, "timeout": 30}
    ).json()

    # Play new chunks immediately
    for chunk in resp["new_chunks"]:
        play_video_chunk(chunk["cdn_url"])
        last_chunk = max(last_chunk, chunk["chunk_id"])

    if resp["status"] == "complete":
        break
```
