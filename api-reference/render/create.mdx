---
title: 'Create Render Job'
api: 'POST /api/render'
description: 'Start a video render job'
---

Start a video render job using `multipart/form-data`. Both character and scene must have status `ready`.

<Note>
  This endpoint requires authentication. Include your API key in the `Authorization: Bearer` header.
</Note>

## Request

Parameters are sent as form fields (not JSON).

<ParamField body="character_id" type="string" required>
  ID of a ready character.
</ParamField>

<ParamField body="scene_id" type="string" required>
  ID of a ready scene.
</ParamField>

<ParamField body="model" type="string" default="V4_Preview">
  Rendering model. Currently only `V4_Preview` is available.
</ParamField>

<ParamField body="background_mode" type="string" default="inpainted">
  `inpainted`, `original`, `solid`, or `transparent`
</ParamField>

<ParamField body="bg_color" type="string">
  RGB string for solid background, e.g., `"0,255,0"`. Only used when `background_mode` is `solid`.
</ParamField>

<ParamField body="resolution" type="string" default="SD">
  Output resolution: `SD` or `HD`.
</ParamField>

<ParamField body="fast" type="boolean" default="false">
  Enable fast mode for faster rendering.
</ParamField>

<ParamField body="output_fps" type="integer">
  Output video frame rate (1-60). Optional — if not set, uses the scene's native FPS.
</ParamField>

<ParamField body="mask_dilation" type="integer" default="12">
  Mask edge dilation in pixels (0-50). Controls how far the character mask extends beyond detected edges.
</ParamField>

<ParamField body="ref_image" type="file">
  Character image file for on-demand render (PNG, JPG). Use instead of `character_id` to skip preprocessing.
</ParamField>

<ParamField body="driving_video" type="file">
  Driving video for on-demand render (MP4). Use instead of `scene_id` to skip preprocessing.
</ParamField>

## Response

<ResponseField name="job_id" type="string">
  Unique identifier for the render job
</ResponseField>

<ResponseField name="status" type="string">
  Initial status will be `queued`
</ResponseField>

<ResponseField name="poll_url" type="string">
  URL to poll for live render chunks (e.g., `/api/render/{job_id}/poll`)
</ResponseField>

<ResponseField name="status_url" type="string">
  URL to check job status (e.g., `/api/render/{job_id}`)
</ResponseField>

<RequestExample>

```bash cURL
curl -X POST "https://apis.viggle.ai/api/render" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d "character_id=char_550e8400e29b41d4" \
  -d "scene_id=scene_660e8400e29b41d4" \
  -d "background_mode=inpainted"
```

```python Python
import requests

response = requests.post(
    "https://apis.viggle.ai/api/render",
    headers={"Authorization": "Bearer YOUR_API_KEY"},
    data={
        "character_id": "char_550e8400e29b41d4",
        "scene_id": "scene_660e8400e29b41d4",
        "background_mode": "inpainted"
    }
)
job = response.json()
print(f"Job ID: {job['job_id']}")
```

```javascript JavaScript
const form = new URLSearchParams();
form.append("character_id", "char_550e8400e29b41d4");
form.append("scene_id", "scene_660e8400e29b41d4");
form.append("background_mode", "inpaint");

const response = await fetch("https://apis.viggle.ai/api/render", {
  method: "POST",
  headers: { "Authorization": `Bearer ${API_KEY}` },
  body: form
});
const job = await response.json();
console.log(`Job ID: ${job.job_id}`);
```

</RequestExample>

<Note>
  **On-Demand Mode**: Instead of pre-creating character and scene assets, you can upload `ref_image` and `driving_video` directly as files. This skips preprocessing and renders in a single request — useful for one-off renders where you don't need to reuse assets.
</Note>

## Next Steps

After creating a render job, you can either:

- **Live Render**: Use [Poll for Chunks](/api-reference/render/poll) to receive video chunks progressively
- **Batch Render**: Use [Get Job Status](/api-reference/render/status) (no auth required) to check progress, then [Download Video](/api-reference/render/download) when complete
- **Cancel**: Use [Cancel Job](/api-reference/render/cancel) to stop a running job
