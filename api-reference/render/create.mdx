---
title: 'Create Render Job'
openapi: 'POST /api/render'
description: 'Start a video render job'
---

Start a video render job using `multipart/form-data`. Both character and scene must have status `ready`.

<Note>
  **Two workflows available:**

  - **On-Demand:** Upload `ref_image` and `driving_video` directly — no preprocessing, no stored assets. Costs 1 credit/second of video plus 1 credit for character creation.
  - **Reusable Assets:** Pre-create characters (1 credit each) and scenes (free) for reuse across renders. Each render costs 1 credit/second. Best for applying the same motion to multiple characters.
</Note>

## Next Steps

After creating a render job, you can either:

- **Live Render**: Use [Poll for Chunks](/api-reference/render/poll) to receive [fMP4](/guides/fmp4-format) video chunks progressively. See the [Live Streaming guide](/guides/live-render) for MSE browser playback and WebSocket examples.
- **Batch Render**: Use [Get Job Status](/api-reference/render/status) (no auth required) to check progress, then [Download Video](/api-reference/render/download) when complete
- **Cancel**: Use [Cancel Job](/api-reference/render/cancel) to stop a running job
