title: 'Create Render Job'
api: 'POST /api/render'
description: 'Start a video render job'
---

Start a video render job using `multipart/form-data`. Both character and scene must have status `ready`.

<Note>
  This endpoint requires authentication. Include your API key in the `Authorization: Bearer` header.
</Note>

## Request

Parameters are sent as form fields (not JSON).

<ParamField body="character_id" type="string" required>
  ID of a ready character.
</ParamField>

<ParamField body="scene_id" type="string" required>
  ID of a ready scene.
</ParamField>

<ParamField body="model" type="string" default="V4">
  Rendering model. Currently only `V4` is available.
</ParamField>

<ParamField body="background_mode" type="string" default="inpaint">
  `inpaint`, `original`, `solid`, or `transparent`
</ParamField>

<ParamField body="bg_color" type="string">
  RGB string for solid background, e.g., `"0,255,0"`. Only used when `background_mode` is `solid`.
</ParamField>

<ParamField body="output_fps" type="integer">
  Output frame rate. Omit to match input video.
</ParamField>

<ParamField body="resolution" type="string" default="original">
  Output resolution: `original`, `720p`, or `1080p`.
</ParamField>

<ParamField body="mask_dilation" type="integer" default="0">
  Pixel dilation for character mask (0-20).
</ParamField>

<ParamField body="fast" type="boolean" default="false">
  Enable parallel processing for ~2x speed.
</ParamField>

## Response

<ResponseField name="job_id" type="string">
  Unique identifier for the render job
</ResponseField>

<ResponseField name="status" type="string">
  Initial status will be `queued`
</ResponseField>

<RequestExample>

```bash cURL
curl -X POST "https://api.viggle.ai/api/render" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d "character_id=char_550e8400e29b41d4" \
  -d "scene_id=scene_660e8400e29b41d4" \
  -d "background_mode=inpaint"
```

```python Python
import requests

response = requests.post(
    "https://api.viggle.ai/api/render",
    headers={"Authorization": "Bearer YOUR_API_KEY"},
    data={
        "character_id": "char_550e8400e29b41d4",
        "scene_id": "scene_660e8400e29b41d4",
        "background_mode": "inpaint"
    }
)
job = response.json()
print(f"Job ID: {job['job_id']}")
```

```javascript JavaScript
const form = new URLSearchParams();
form.append("character_id", "char_550e8400e29b41d4");
form.append("scene_id", "scene_660e8400e29b41d4");
form.append("background_mode", "inpaint");

const response = await fetch("https://api.viggle.ai/api/render", {
  method: "POST",
  headers: { "Authorization": `Bearer ${API_KEY}` },
  body: form
});
const job = await response.json();
console.log(`Job ID: ${job.job_id}`);
```

</RequestExample>

</ResponseExample>

## Next Steps

After creating a render job, you can either:

- **Live Render**: Use [Poll for Chunks](/api-reference/render/poll) to receive video chunks progressively
- **Batch Render**: Use [Get Job Status](/api-reference/render/status) (no auth required) to check progress, then [Download Video](/api-reference/render/download) when complete
- **Cancel**: Use [Cancel Job](/api-reference/render/cancel) to stop a running job
