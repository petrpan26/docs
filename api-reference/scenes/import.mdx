---
title: 'Import Template'
api: 'POST /api/scenes/import'
description: 'Import a scene from a Viggle template'
---

Import a scene from a Viggle template. Supports two template types:

- **Multi-person templates** — contain pre-tracked segments for each person in the video, enabling multi-character rendering
- **Video templates** — plain video scenes with no person tracking, used for single-character rendering

The API automatically detects the template type based on the UUID.

<Info>
  **Multi-Character Scenes**

  Multi-person templates contain person segments with unique UUIDs. Use these UUIDs in [Multi-Character Render](/api-reference/render/multi-character) to map different avatars to each person.

  Video templates have no segments — use them with [Create Render Job](/api-reference/render/create) for single-character rendering.
</Info>

## Request

<ParamField body="template_id" type="string" required>
  Viggle template UUID from your library. Can be either a multi-person template or a video template.
</ParamField>

<ParamField body="name" type="string">
  Optional display name for the scene.
</ParamField>

## Response

<ResponseField name="id" type="string">
  Unique identifier for the imported scene
</ResponseField>

<ResponseField name="status" type="string">
  Initial status will be `pending`
</ResponseField>

<ResponseField name="credits_reserved" type="integer">
  Number of credits reserved
</ResponseField>

<ResponseField name="segments" type="array">
  Array of tracked person segments. Present for multi-person templates, empty for video templates. Each segment has:

  <Expandable title="Segment object">
    <ResponseField name="uuid" type="string">
      Unique identifier for this person (e.g., `person_a1b2c3d4`)
    </ResponseField>
    <ResponseField name="type" type="string">
      Segment type (typically `human`)
    </ResponseField>
  </Expandable>
</ResponseField>

## Template Types

### Multi-Person Template

Multi-person templates include tracked person segments. The response contains a `segments` array with UUIDs for each person. Use with [Multi-Character Render](/api-reference/render/multi-character).

<RequestExample>

```bash cURL (Multi-Person)
curl -X POST "https://apis.viggle.ai/api/scenes/import" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "template_id": "multi_person_template_uuid",
    "name": "Group Dance"
  }'
```

```python Python (Multi-Person)
import requests

response = requests.post(
    "https://apis.viggle.ai/api/scenes/import",
    headers={
        "Authorization": "Bearer YOUR_API_KEY",
        "Content-Type": "application/json"
    },
    json={
        "template_id": "multi_person_template_uuid",
        "name": "Group Dance"
    }
)
data = response.json()

# Get segment UUIDs for character mapping
for segment in data["segments"]:
    print(f"Person: {segment['uuid']}")
```

```javascript JavaScript (Multi-Person)
const response = await fetch("https://apis.viggle.ai/api/scenes/import", {
  method: "POST",
  headers: {
    "Authorization": `Bearer ${API_KEY}`,
    "Content-Type": "application/json"
  },
  body: JSON.stringify({
    template_id: "multi_person_template_uuid",
    name: "Group Dance"
  })
});
const data = await response.json();

// Get segment UUIDs for character mapping
data.segments.forEach(segment => {
  console.log(`Person: ${segment.uuid}`);
});
```

</RequestExample>

<ResponseExample>

```json Multi-Person Template
{
  "id": "scene_770e8400e29b41d4",
  "status": "pending",
  "credits_reserved": 15,
  "segments": [
    { "uuid": "person_a1b2c3d4", "type": "human" },
    { "uuid": "person_e5f6g7h8", "type": "human" }
  ]
}
```

```json Video Template
{
  "id": "scene_880f9500f30c52e5",
  "status": "pending",
  "credits_reserved": 10,
  "segments": []
}
```

</ResponseExample>

### Video Template

Video templates are plain video scenes with no person tracking data. The response will have an empty `segments` array. Use with [Create Render Job](/api-reference/render/create) for single-character rendering.

```bash
curl -X POST "https://apis.viggle.ai/api/scenes/import" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "template_id": "video_template_uuid",
    "name": "Solo Dance"
  }'
```

```python
import requests

response = requests.post(
    "https://apis.viggle.ai/api/scenes/import",
    headers={
        "Authorization": "Bearer YOUR_API_KEY",
        "Content-Type": "application/json"
    },
    json={
        "template_id": "video_template_uuid",
        "name": "Solo Dance"
    }
)
scene = response.json()
scene_id = scene["id"]

# No segments — use with single-character render
job = requests.post(
    "https://apis.viggle.ai/api/render",
    headers={"Authorization": "Bearer YOUR_API_KEY"},
    data={
        "character_id": "char_xxx",
        "scene_id": scene_id
    }
).json()
print(f"Job ID: {job['job_id']}")
```

## Using Segment UUIDs

For multi-person templates, the `segments` array contains UUIDs for each tracked person. Use these UUIDs in the `character_mapping` parameter when [creating a multi-character render](/api-reference/render/multi-character).

```json
{
  "scene_id": "scene_770e8400e29b41d4",
  "character_mapping": {
    "person_a1b2c3d4": "char_alice",
    "person_e5f6g7h8": "char_bob"
  }
}
```
