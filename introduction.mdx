Generate AI-powered character animation videos with the Viggle API. Bring your own driving video or [**import from Viggle's rich library**](#import-templates) of popular video templates — dances, memes, and more. There are three ways to render: [**On-Demand**](#on-demand-rendering) — send an image and video in one request, [**Preprocessed**](#preprocess-for-speed) — pre-create assets for ~3x faster renders, and [**Live**](#live-rendering) — stream chunks in real-time.

## On-Demand Rendering

The simplest way to get started. Send a character image and a driving video in a single request — the API handles everything and returns a rendered video. No setup or preprocessing needed.

<CodeGroup>

```python Python
import requests, time

API_KEY = "YOUR_API_KEY"
BASE = "https://apis.viggle.ai"
headers = {"Authorization": f"Bearer {API_KEY}"}

# Render — upload image + video directly
job = requests.post(f"{BASE}/api/render", headers=headers, files={
    "ref_image": open("character.png", "rb"),
    "driving_video": open("dance.mp4", "rb"),
}).json()

# Poll until complete
while requests.get(f"{BASE}/api/render/{job['job_id']}").json()["status"] != "complete":
    time.sleep(5)

# Download
video = requests.get(f"{BASE}/api/render/{job['job_id']}/download", headers=headers)
open("output.mp4", "wb").write(video.content)
```

## Preprocess for Speed

For production workflows, pre-create characters and scenes ahead of time to **cut render latency by ~3x**. The API extracts character embeddings and motion data once, then reuses them instantly across unlimited renders — no re-uploading or re-processing needed.

```python
# 1. Create character once (free, ~5s to process)
char = requests.post(f"{BASE}/api/characters/preprocess", headers=headers,
    files={"image": open("character.png", "rb")},
    data={"name": "My Character"}
).json()

# 2. Create scene once (free, ~20s to process)
scene = requests.post(f"{BASE}/api/scenes/preprocess", headers=headers,
    files={"video": open("dance.mp4", "rb")},
    data={"name": "Dance"}
).json()

# 3. Render with stored IDs — skips preprocessing, ~3x faster
job = requests.post(f"{BASE}/api/render", headers=headers, data={
    "character_id": char["id"],
    "scene_id": scene["id"],
}).json()
```

## Live Rendering

Instead of waiting for the full video, stream fMP4 chunks as they render. Use long-polling to receive each chunk the moment it's ready — ideal for real-time previews, progressive playback, and streaming applications.

```python
last_chunk = -1
while True:
    resp = requests.get(f"{BASE}/api/render/{job_id}/poll",
        headers=headers,
        params={"last_chunk": last_chunk, "timeout": 30}
    ).json()

    for chunk in resp["new_chunks"]:
        print(f"Chunk {chunk['chunk_id']}: {chunk['cdn_url']}")
        last_chunk = chunk["chunk_id"]

    if resp["status"] == "complete":
        break
```
